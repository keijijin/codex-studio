## 1. インストールと起動

### 開発実行

```bash
pnpm install
pnpm dev
```

### パッケージ済みアプリ

GitHub Releases のインストーラ（macOS / Windows / Linux）を利用します。  
手順の詳細は [docs/release.md](../release.md) を参照してください。

---

## 2. 最初のセットアップ

1. アプリを起動する
2. **Open Folder** でプロジェクト（ワークスペース）を開く
3. **設定** を開き、LLM を登録する

| プロバイダ | 設定 |
|-----------|------|
| OpenAI | API キー + モデル |
| Anthropic | API キー + モデル |
| Ollama | Base URL（既定 `http://localhost:11434`）+ ローカルモデル |

環境変数 `OPENAI_API_KEY` / `ANTHROPIC_API_KEY` / `OLLAMA_BASE_URL` も利用できます。

4. 右側 **AI Chat** で Ask または Agent を選んで質問する

---

## 3. 基本操作（IDE）

### 3.1 エディタ

- Explorer からファイルを開く
- `.md` はプレビュー / 編集を切り替え可能
- 保存: macOS `Cmd+S` / Windows `Ctrl+S`
- タブをすべて閉じる: macOS `Cmd+Alt+W` / Windows `Ctrl+Alt+W`

### 3.2 検索・ターミナル

- サイドバー **Search** でワークスペース横断検索（ripgrep）
- 下部パネルで組み込みターミナル（ログインシェル相当の環境を起動）

### 3.3 Ask / Agent

| モード | 用途 |
|--------|------|
| **Ask** | 質問・説明。`@ファイル名` で添付可 |
| **Agent** | 調査・編集・Shell。ツールを使って作業 |

Agent でファイルを書くときは、既定では **承認ダイアログ** で Diff を確認してから適用します。

---

## 4. 設定（安全と品質）

設定画面の主な項目です。

| 項目 | 説明 |
|------|------|
| YOLO モード | ON で書込・Shell を承認なし実行（危険。信頼できる環境のみ） |
| Permission | 読取 / 編集 / Shell / ネットワークごとに `allow` / `ask` / `deny` |
| 最大イテレーション | Agent のツールループ上限（既定 100） |
| Compact 閾値 | 履歴の推定トークン超過で自動圧縮 |
| 並列 Subagent 上限 | `Task` の同時実行数（1〜8） |
| Auto Memory | Agent 完了後に memory へ要約追記（オプトイン） |

**YOLO が ON のときは Permission より優先**されます。

## 5. エージェント拡張（Phase A〜D）

### 5.1 Skills（スラッシュコマンド）

再利用可能な手順をファイルに置き、`/名前` で起動します。

**配置**

```
{workspace}/.codex/skills/<name>/SKILL.md
~/.codex-studio/skills/<name>/SKILL.md   # 全プロジェクト共有（同名はワークスペース優先）
```

**例（このリポジトリ同梱）**

| コマンド | 用途 |
|----------|------|
| `/review` | 変更レビュー |
| `/security` | セキュリティ観点 |
| `/explain` | モジュール説明 |

入力欄で `/` と打つと候補が出ます。Ask / Agent どちらでも使えます。

**SKILL.md の形**

```markdown
---
name: review
description: 候補に出る短い説明
argument-hint: 任意引数のヒント
---

ここに手順を書く
```

### 5.2 プロジェクト常駐コンテキスト

ワークスペース直下にあれば Rules の後に system へ自動注入されます。

| ファイル | 用途 |
|----------|------|
| `CODEX.md` | 推奨 |
| `CLAUDE.md` | Claude Code 互換 |
| `AGENTS.md` | 汎用エージェント向け |

### 5.3 Rules

| 場所 | 用途 |
|------|------|
| `.codex/rules/*.md` | プロジェクト Rules |
| `.cursor/rules/*.{md,mdc}` | Cursor 互換 |
| `~/.codex-studio/rules/*.md` | グローバル |

設定の **Rules** タブからも追加できます。

### 5.4 Compact

- **自動**: 設定の Compact 閾値超過時（Agent 実行中）
- **手動**: AI Chat の **Compact** ボタン

### 5.5 Hooks（イベント駆動）

`.codex/hooks.json` で保存時・Agent 完了時に Shell または Skill を起動できます。

```bash
cp .codex/hooks.example.json .codex/hooks.json
```

主なフィールド:

| フィールド | 内容 |
|------------|------|
| `event` | `onFileSave` / `onAgentComplete` |
| `paths` / `exclude` | glob（保存時） |
| `action` | `{ "type": "shell", "command": "..." }` または `{ "type": "skill", "skill": "review" }` |
| 変数 | `${file}` `${relativePath}` `${workspace}` `${sessionId}` |

再入防止とクールダウン付きです。Main プロセス更新後は `pnpm dev` の再起動が必要です。

### 5.6 Headless CLI

Electron なしで同じ agent-core を実行します。

```bash
# Agent
pnpm cli -- agent "README を要約して" -w .

# 別プロバイダ
pnpm cli -- agent "/review" -w . -p anthropic
pnpm cli -- agent "一覧して" -w . -p ollama -m llama3.2

# JSON 出力
pnpm cli -- agent "…" -w . --json
```

| オプション | 説明 |
|------------|------|
| `-w` | ワークスペース（既定: cwd） |
| `-p` | `openai` / `anthropic` / `ollama` |
| `-m` | モデル ID |
| `--profile` | `readonly`（既定）/ `ask` / `allow` |
| `--yolo` | 全ツール allow |
| `--json` | 結果を JSON |

`codex-studio` 単体は PATH に入りません。`pnpm cli` または `pnpm exec codex-studio` を使ってください。

**注意**: OpenAI で `429 quota` が出た場合は課金・枠を確認するか、`-p anthropic` / `-p ollama` に切り替えてください。

### 5.7 Subagents（Task）

Agent が `Task` ツールで読取専用サブエージェントを並列起動します。

- 子が使えるツール: Read / Grep / Glob のみ
- 入れ子 Task は不可
- 並列数は設定の「並列 Subagent 上限」

チャットでは「認証と IPC を別々に調べてまとめて」のように依頼すると、モデルが複数 Task を使う想定です。

### 5.8 Auto Memory / MemoryAppend

| 方法 | 説明 |
|------|------|
| Auto Memory（設定） | 完了後に `.codex/memory/MEMORY.md` へ要約追記 |
| MemoryAppend ツール | Agent が規約・決定を明示追記（承認対象） |

次回セッションで memory が system に注入されます。

### 5.9 WebSearch

`WebSearch` ツールで公開 Web を検索します（API キー不要）。  
権限カテゴリは **network**（設定で deny 可）。

### 5.10 Agent Teams（ローカル）

複数役割を並列実行し、共有ボードに書いてから統合します。

**配置**

```
.codex/teams/<id>/team.json
.codex/teams/<id>/BOARD.md
```

**サンプル**: `review-squad`（correctness / security / lead）

```bash
pnpm cli -- team list -w .
pnpm cli -- team run review-squad "IPC と権限をレビュー" -w .
```

Agent チャットでは:

```text
/team review-squad IPC と権限をレビュー
```

またはモデルが `Team` ツールを呼び出します。  
チーム実行は読取中心です（書込・Shell はチーム側では禁止）。

Cloud VM / ブラウザ遠隔は **未実装** です（[Phase D ゲート](../09-PhaseD-設計ゲート.md)）。

## 6. 推奨ワークフロー例

### コードレビュー

1. 変更をコミットまたは作業ツリーに残す
2. Agent で `/review` または `/team review-squad 最近の変更をレビュー`
3. 指摘を確認し、必要なら Agent に修正依頼（承認して適用）

### 調査だけ（書込なし）

1. Permission の edit / shell を `deny`、または CLI `--profile readonly`
2. Ask または Agent で質問

### CI / スクリプトから

```bash
export OPENAI_API_KEY=...   # または ANTHROPIC / Ollama
pnpm cli -- agent "テスト失敗の原因を要約" -w . --profile readonly --json
```

## 7. トラブルシューティング

| 症状 | 対処 |
|------|------|
| Agent が動かない | `pnpm dev` を再起動（Main 更新のため） |
| API キーエラー | 設定または環境変数を確認 |
| `429` / quota | 課金・枠を確認。`-p anthropic` や `-p ollama` を試す |
| `codex-studio: command not found` | `pnpm cli -- …` または `pnpm exec codex-studio …` |
| CLI がヘルプだけ出る | `pnpm cli -- agent "…"` のように `--` の後に引数 |
| Hooks が動かない | `.codex/hooks.json` と再起動を確認 |
| 日本語なのに英語で返る | 指示文に日本語を含める（言語検出あり）。アプリ再起動 |
| Ollama 接続失敗 | `ollama serve` と Base URL を確認 |
