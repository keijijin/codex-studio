## 1. 目的

OpenAI / Anthropic / Ollama を統合したうえで、**実行ごとに最適なプロバイダ・モデルを選ぶ**（手動固定ではなくルーティング）仕組みを導入する。

### ゴール

- ユーザーは「Auto」モードを選べる
- 失敗（429 / 5xx / 接続不可）時に次候補へフォールバックできる
- タスク種別（軽い質問 / コーディング Agent / 長文要約）に応じて候補を変えられる
- 選択結果が UI / ログで説明可能（なぜそのモデルか）

### 非目標（当面やらない）

- 機械学習ベースの高度なルーター学習・オンライン学習
- プロバイダ横断のストリーム途中ホットスワップ（同一リクエスト内で途中切替）
- コスト課金ダッシュボードの本格実装（簡易メトリクスのみ）
- 新プロバイダ追加（Gemini 等）— 既存 3 系のルーティングを先に固める

---

## 2. 現状とギャップ

| 項目 | 現状 | ギャップ |
|------|------|----------|
| プロバイダ統合 | `openai` / `anthropic` / `ollama` アダプタ済み | なし（土台あり） |
| モデル選択 | 設定の `defaultProvider` + 単一モデル | 会話ごと・送信ごとの切替 UI が弱い |
| 実行時解決 | `getLlmRuntimeConfig()` が固定 1 本 | 候補列・フォールバックなし |
| エラー時 | エラー表示で終了 | 自動リトライ / 次プロバイダへ遷移なし |
| 最適選択 | なし | ルールベース or ヒューリスティック未実装 |
| 透明性 | StatusBar 等でモデル表示はある程度 | 「Auto が選んだ理由」の表示なし |

---

## 3. 方針

| 方針 | 内容 |
|------|------|
| **単一エントリ** | Chat / Agent / CLI / Hooks / Team が同じ `ModelRouter` を通る |
| **オプトイン** | 既定は現行どおり固定モデル。`routing.mode = 'auto'` で有効化 |
| **説明可能性** | 選択結果に `reason` / `candidatesTried` を付与 |
| **安全** | API キー未設定の候補はスキップ。Ollama 未起動もスキップ |
| **段階導入** | R1 フォールバック → R2 ヒューリスティック → R3 UI → R4 高度化 |

### ルーティングモード

```ts
type RoutingMode = 'fixed' | 'auto' | 'fallback-only'
```

| モード | 挙動 |
|--------|------|
| `fixed` | 現行どおり。設定の provider/model のみ（既定） |
| `fallback-only` | 既定モデルで開始。失敗時のみ候補チェーンを辿る |
| `auto` | タスク分類 → 候補順を決定 → 成功するまで試行 |

## 4. アーキテクチャ

```
┌─────────────┐   ┌──────────────┐   ┌─────────────────┐
│ Chat/Agent  │──▶│ ModelRouter  │──▶│ getProvider…()  │
│ CLI / Hooks │   │  classify    │   │ stream/chat     │
└─────────────┘   │  rank        │   └─────────────────┘
                  │  fallback    │
                  └──────┬───────┘
                         │
                  ┌──────▼───────┐
                  │ AppSettings  │
                  │ .routing     │
                  └──────────────┘
```

### 新規モジュール案

| パス | 役割 |
|------|------|
| `packages/llm-adapters/src/router/` | 分類・順位付け・試行結果型（Electron 非依存） |
| `packages/app/.../llm-config.ts` | 設定 → `RoutingDecision` 解決の拡張 |
| `packages/shared/src/types.ts` | `routing` 設定スキーマ |
| Renderer: Settings / AIPanel | Auto 切替・選択理由表示 |

### 中核型（案）

```ts
type TaskKind =
  | 'chat_simple'
  | 'chat_long'
  | 'agent_code'
  | 'agent_explore'
  | 'team'
  | 'unknown'

interface ModelCandidate {
  provider: LLMProviderId
  model: string
  priority: number
}

interface RoutingDecision {
  mode: RoutingMode
  selected: ModelCandidate
  reason: string
  queue: ModelCandidate[]  // フォールバック順
  taskKind?: TaskKind
}
```

### 挿入ポイント

1. `getLlmRuntimeConfig` → `resolveRoutingDecision(settings, { mode, prompt, isTeam })`
2. `chat.ts` / `agent.ts` / CLI: ストリーム開始前に decision を確定
3. LLM 呼び出し失敗時: `isRetryableError` → 次候補で再構築（**同一 user turn 内**）
4. 成功時: `session` または stream event に `routedModel` / `reason` を付与

## 5. フェーズ計画

```
R1: フォールバック基盤（設定 + エラー時切替）     〜 3–5 日
R2: ヒューリスティック自動選択（タスク種別）       〜 3–5 日
R3: UI / CLI / 透明性（Auto・理由表示）            〜 3–5 日
R4: 高度化（コスト上限・ヘルスチェック・学習）     必要時
```

### R1 — フォールバック基盤

**ゴール**: `fallback-only` で 429 / 接続失敗時に次プロバイダへ自動切替

| # | 成果物 | 完了条件 |
|---|--------|----------|
| R1-1 | `routing` 設定スキーマ | `mode` / `fallbackChain` / 候補ごとの model |
| R1-2 | `ModelRouter`（固定 + チェーン） | fixed / fallback-only が動く |
| R1-3 | 再試行可能エラー判定 | 429, 5xx, ECONNREFUSED, Ollama down |
| R1-4 | Chat / Agent への組込 | 失敗時に次候補で 1 ターン再実行（履歴二重投稿なし） |
| R1-5 | 単体テスト | チェーン順・スキップ（キーなし）・非再試行エラー |

**既定フォールバック例**

```
openai:gpt-4o → anthropic:claude-sonnet → ollama:qwen2.5-coder:14b
```

（キー未設定・到達不可はスキップ）

### R2 — ヒューリスティック自動選択

**ゴール**: `auto` でプロンプト／モードから候補順を決める

| # | 成果物 | 完了条件 |
|---|--------|----------|
| R2-1 | `classifyTaskKind` | Agent / Team / 長さ / キーワードで分類（LLM 呼び出しなし） |
| R2-2 | プロファイル表 | 種別ごとの候補優先度（設定で上書き可） |
| R2-3 | 利用可能性フィルタ | API キー有無・Ollama ping（短タイムアウト） |
| R2-4 | Chat / Agent / CLI 統一 | 同じ分類・順位付けを使用 |
| R2-5 | テスト | 代表プロンプトでの順位スナップショット |

**初期ヒューリスティック案**

| TaskKind | 優先（例） |
|----------|------------|
| `chat_simple` | Ollama 軽量 → OpenAI mini → Anthropic Haiku |
| `agent_code` | Anthropic Sonnet → OpenAI → Ollama Coder |
| `agent_explore` | 高速・安価寄り（mini / Haiku / local） |
| `team` | 既定 Agent モデル相当（品質優先） |
| `chat_long` | 長コンテキスト強いモデル優先 |

※ 具体的 model ID は設定の `profiles` で差し替え可能にする。

### R3 — UI / CLI / 透明性

**ゴール**: ユーザーが Auto を使え、選択理由が見える

| # | 成果物 | 完了条件 |
|---|--------|----------|
| R3-1 | 設定 UI | Routing mode / fallback チェーン編集 |
| R3-2 | AIPanel クイック切替 | Fixed モデル or Auto（送信バー付近） |
| R3-3 | Stream / Status 表示 | 「Anthropic · …（Auto: agent_code）」 |
| R3-4 | CLI | `--routing auto|fixed|fallback` と理由の stderr/json |
| R3-5 | 操作マニュアル更新 | Auto / フォールバックの使い方 |

### R4 — 高度化（任意・後続）

| # | 内容 | 備考 |
|---|------|------|
| R4-1 | Ollama ヘルス常駐キャッシュ | 毎回 ping しない |

## 6. 設定スキーマ（案）

`AppSettings.models` を拡張、または隣接キー `routing` を追加:

```ts
routing: {
  mode: 'fixed' | 'auto' | 'fallback-only'
  /** 失敗時に辿る候補（provider+model）。空ならビルトイン既定 */
  fallbackChain: Array<{ provider: LLMProviderId; model: string }>
  /** auto 時のプロファイル上書き */
  profiles?: Partial<Record<TaskKind, Array<{ provider: LLMProviderId; model: string }>>>
  /** 1 ターンあたり最大試行数 */
  maxAttempts: number  // default 3
}
```

既存の `defaultProvider` / `defaultChatModel` / `defaultAgentModel` は **fixed のソース**および **auto のヒント**として維持する。

---

## 7. エラー再試行の注意点

| 論点 | 方針 |
|------|------|
| ユーザーメッセージ二重投稿 | 最初の 1 回だけ `addMessage`。再試行は assistant 途中結果を破棄 |
| ツール実行途中の失敗 | **ツール開始前**の LLM 失敗のみフォールバック。ツール後はフォールバックしない（副作用回避） |
| 部分ストリーム表示 | 切替時に「別モデルで再試行中…」イベントを送る |
| Abort | ユーザー Cancel はフォールバックしない |

## 9. リスクと緩和

| リスク | 緩和 |
|--------|------|
| 予期せぬプロバイダ課金 | maxAttempts・明示 Opt-in・安価候補を先に置ける設定 |
| ツール途中フォールバックの破壊 | R1 では「初回 LLM 応答前」のみ再試行 |
| 分類ミス | reason 表示 + 手動 fixed へ即戻し |
| レイテンシ増（ping 等） | 並列可用性チェック + 短い timeout（例 300ms） |

## 10. 成功指標

- Auto 有効時、OpenAI 429 でも Anthropic / Ollama で同一プロンプトが完走する
- 固定モード利用者の既存挙動が変わらない（回帰ゼロ）
- 選択理由が UI または CLI で確認できる

## 11. 実装順序（推奨）

1. 型・設定・`ModelRouter` 骨格（R1-1, R1-2）
2. Chat/Agent の再試行組込（R1-3, R1-4）+ テスト
3. `classifyTaskKind` + profiles（R2）
4. Settings / AIPanel / マニュアル（R3）
5. dogfooding 後に R4 を判断

## 12. 課金最適化ティア（2026-07-19）

### 方針

すべてを最強モデルに投げず、**タスク複雑度・推定トークン量**で開始ティアを決め、失敗時のみ上位へ昇格する。

| ティア | OpenAI | Anthropic | xAI (Grok) |
|--------|--------|-----------|------------|
| lite | `gpt-5.4-nano` | `claude-haiku-4-5` | `grok-4-1-fast-non-reasoning` |
| standard | `gpt-5.4-mini` | `claude-sonnet-4-6` | `grok-4.3` |
| premium | `gpt-5.5` | `claude-opus-4-8` | `grok-4.5` |

### 分類

- `chat_simple` → lite
- `agent_explore` / 軽いコード → lite〜standard
- 通常 Agent コード → standard（Claude Sonnet 優先）
- `agent_hard` / team / アーキテクチャ・RCA・セキュリティ語 → premium

### カタログ自動更新

- `packages/llm-adapters/src/router/catalog.ts` にエイリアス表
- `modelCatalogService` が各 API の `listModels` 結果で解決し、`model-catalog.json` に 24h TTL で保存
- MODELS_LIST / Auto ルーティング時にバックグラウンド更新
