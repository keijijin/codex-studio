# 開発プラン：Codex Studio

| 項目 | 内容 |
|------|------|
| 版 | 1.0 |
| 作成日 | 2026-07-15 |
| 期間 | 2026-07-15 〜 2026-10-15（MVP 3ヶ月） |

---

## 1. 開発方針

| 方針 | 内容 |
|------|------|
| **MVP First** | Phase 1 で「Agent が単一タスクを完遂できる」体験に集中 |
| **Vertical Slice** | 各スプリントで動く縦切り機能を deliver |
| **Dogfooding** | 自プロジェクトの開発に Codex Studio 自体を使用（Sprint 3〜） |
| **Feature Flag** | 未完成機能はフラグで隠蔽 |
| **CI/CD 早期構築** | Sprint 0 でパイプライン整備 |

---

## 2. フェーズ概要

```
Phase 0: 基盤構築          Week 1-2
Phase 1: エディタ + チャット  Week 3-5
Phase 2: Agent + ツール      Week 6-9
Phase 3: 仕上げ + α リリース  Week 10-12
Phase 4: 拡張（Post-MVP）    Week 13+
```

---

## 5. チーム体制

```
PM ───────────── 全体調整、優先度、ステークホルダー
  │
  ├── FE Lead ─── UI/UX、Renderer、Monaco
  ├── BE Lead ─── Main、Agent、Indexer、IPC
  ├── ML Eng ─── LLM Adapters、Prompt、評価
  └── DevOps/QA ─ CI/CD、E2E、リリース
```

### 5.1 コミュニケーション

|----------|------|------|

| Sprint Review | 2週間毎 | デモ |
| Retrospective | 2週間毎 | 改善 |
| Architecture Review | 月1 | 設計判断 |

## 10. 成功の定義（MVP）

1. 社内開発者 10 名が 1 週間以上日常使用
2. Agent タスク成功率 60% 以上（定義済みベンチマーク 20 件）
3. 重大バグ（データ損失、セキュリティ）ゼロ
4. 主要操作の E2E テスト 100% パス
