## 1. エグゼクティブサマリー

本プロジェクトは、Cursor に代表される **AI ネイティブな開発環境** を自社/自チーム向けに構築することを目的とする。開発者がコードベースの文脈を保ったまま、チャット・エージェント・インライン補完を通じて生産性を向上させる **デスクトップ + クラウド連携型 IDE** を提供する。

---

## 2. 背景と課題

### 2.1 背景

- LLM の能力向上により、コード生成・リファクタリング・デバッグ支援が実用段階に入った
- 既存 IDE へのプラグイン追加では、コンテキスト管理・ツール実行・セッション管理の統合体験が弱い
- 社内コードベース・規約・セキュリティ要件を満たす **カスタマイズ可能な AI 開発環境** の需要が増加

### 2.2 解決したい課題

| 課題 | 現状 | 目指す姿 |
|------|------|----------|
| コンテキスト不足 | チャットにファイルを手動貼付 | ワークスペース全体を索引化し自動参照 |
| 変更の安全性 | AI 提案をそのまま適用 | Diff プレビュー・部分適用・ロールバック |
| ツール連携 | 外部 API を個別実装 | MCP 標準で拡張可能なツールエコシステム |
| 運用・監査 | 利用ログが分散 | セッション・ツール実行・モデル利用の一元記録 |
| オフライン/オンプレ | クラウド SaaS 依存 | ローカルモデル・自社 API エンドポイント対応 |

---

## 3. プロジェクト目的

1. **開発者体験（DX）の向上**：自然言語とコード操作のシームレスな統合
2. **エージェント自律実行**：ファイル操作・ターミナル・検索等を安全に委譲
3. **拡張性**：MCP・Skills・Rules による組織固有ワークフロー
4. **セキュリティとガバナンス**：権限モデル・監査ログ・シークレット管理

---

## 4. スコープ

### 4.1 MVP（Phase 1）に含める

- Electron ベースのデスクトップアプリ（VS Code 互換エディタコア）
- サイドバー AI チャット（ストリーミング応答）
- Agent モード（ツール呼び出しループ）
- 基本ツール：Read / Write / Grep / Shell / Glob
- ワークスペース索引（ファイルツリー + 全文検索）
- Diff プレビューと適用
- OpenAI / Anthropic / ローカル（Ollama）対応
- セッション履歴の永続化

### 4.2 Phase 2 以降

- MCP サーバー連携
- インライン補完（Tab 補完）
- クラウドエージェント（リモート VM 実行）
- マルチユーザー・チーム共有 Rules
- CI/CD 連携（PR レビュー Bot）
- モバイル/Web クライアント（閲覧・軽量チャット）

### 4.3 スコープ外（初期）

- 自前 LLM の学習・ファインチューニング
- 完全な Git ホスティング機能
- ノーコードアプリビルダー

## 8. リスクと対策

| リスク | 影響 | 対策 |
|--------|------|------|
| LLM API コスト増 | 高 | キャッシュ、小モデルルーティング、利用上限 |
| エージェントの誤操作 | 高 | サンドボックス、確認ダイアログ、Git 連携 |
| セキュリティ侵害 | 高 | 権限最小化、シークレットスキャン、監査 |
| 開発遅延 | 中 | MVP スコープ固定、Monaco/Electron 活用 |
| 競合の機能追従 | 中 | MCP/Plugin エコシステムで差別化 |
